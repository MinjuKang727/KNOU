varName
c  language >> 공백X
int >> 예약어X
func#1  >> 특수 기호X

C 언어의 설명문(comment)에 대한 설명으로 옳은 것은?
'//'로 설명문을 시작하여 다시 '//'가 나오면 설명문을 마친다.  >> //(x) -> /*로 시작하여 */로 마침
'#'로 시작하는 행은 그 행의 끝까지 설명문이다. >> #(x) -> //(o)
'//' 기호 이후의 문장은 그 행의 끝까지 설명문으로 취급한다.
설명문도 문법이 있어 올바른 문법에 따라 작성해야 한다.  >> 설명문은 별도의 문법 없음

다음 중 C 언어의 문장에 대한 설명으로 옳은 것은?
선언문은 함수 안에만 작성할 수 있다.  >> 함수 밖에서도 선언 가능(ex. 전역 변수) 
복합문은 한 개 이상의 문장을 { }로 묶은 것으로, 내부에 지역변수를 포함할 수 있다. (o)
모든 문장은 마침표('.')으로 끝마친다.  >> 마침표('.')(x) -> 세미콜론(';')(o)
실행문은 함수 밖에만 작성할 수 있다.  >> 실행문은 함수 안에서만 작성 가능
a와 b가 int형 변수일 때, 다음 문장 중 올바른 문장은?
printf('%d', a);  >> 문자열은 작은따옴표('')(x) -> 큰따옴표("")
printf("%f\n", b);  >> int형 양식 변환기호는 %d/ %f는 float 또는 double형
scanf("%d", &a);
scanf("%d", b);  >>  scanf("%d", &b);
현재 사용중인 디렉토리에서 header.h라는 이름의 헤더파일을 포함하게 하는 선행처리기 지시어를 작성하라.
#include "header.h"
